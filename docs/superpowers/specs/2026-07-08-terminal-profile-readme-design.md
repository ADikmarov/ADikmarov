## Цель

Заменить шаблонный профильный README на цельный «терминальный» профиль,
который впечатляет и рекрутёров (аккуратность, информативность), и
разработчиков (видно, что генерация самописная). Контент автообновляется
ежедневно через GitHub Actions.

## Что убираем

- Счётчик просмотров komarev.
- Мёртвый streak-виджет (herokuapp).
- Сетку из ~30 иконок технологий.
- Строку «I'm currently learning AWS».

## Общий вид

README — вертикальная последовательность из четырёх SVG-блоков, оформленных
как одно окно терминала: общий тёмный фон, скруглённые углы, кнопки macOS в
шапке первого блока. Палитра — Catppuccin Mocha, единая для всех блоков.
Терминал самодостаточно тёмный — одинаково выглядит в светлой и тёмной теме
GitHub (без `<picture>`/prefers-color-scheme).

Внизу README обычным текстом: строка ссылок (Instagram, Dribbble, LeetCode)
и подпись «generated daily by GitHub Actions» со ссылкой на workflow.

### 3. `assets/activity.svg` — последняя активность

Оформление под `git log --oneline --graph`: последние 5–7 публичных событий
(пуши, PR, релизы, созданные репо) с относительными датами («2 days ago»).
Источник — GitHub Events API.

### 4. `assets/langs.svg` — языки

Оформление под вывод условной `stats --langs`: ASCII-бары распределения
языков по публичным репозиториям. Правила:

- Проценты округляются до целых.
- Go и Python не отрезаются фильтром топ-N: показываются всегда со своей
  реальной долей, даже если она мала.
- Сумма после округления корректируется, чтобы не было «101%».

## Архитектура

```
README.md              ← ссылки на SVG + текстовый футер
assets/                ← сгенерированные SVG (коммитятся в репо)
scripts/
  render/              ← общий SVG-рендер: рамка окна, палитра, встроенный
                          моноширинный шрифт (base64, ~20-30 КБ)
  sections/            ← модуль на блок: header, neofetch, activity, langs
.github/workflows/
  update-readme.yml
```

- Язык: TypeScript (Node.js), запуск `npm run generate`.
- Данные: GitHub API, авторизация встроенным `GITHUB_TOKEN` — секретов не
  требуется.
- Шрифт встроен в SVG base64 — одинаковый вид у всех посетителей.
- Анимации — только CSS/SMIL внутри SVG (в `<img>` не работают JS и hover).

## Workflow

`update-readme.yml`: cron раз в день (утро Europe/Amsterdam) +
`workflow_dispatch`. Шаги: checkout → Node → `npm run generate` → если SVG
изменились, коммит от github-actions bot (`chore: refresh profile (YYYY-MM-DD)`).
Нет изменений — нет коммита.

## Обработка ошибок

- API недоступен/невалидный ответ → скрипт падает, workflow краснеет,
  коммита нет: в профиле остаётся предыдущая версия.
- Блоки генерируются независимо: сбой одного не блокирует остальные.
- Пустые состояния: нет свежих событий → показываются последние доступные.

## Тестирование

- Локальный запуск `npm run generate` — просмотр SVG в браузере до пуша.
- Юнит-тесты на хрупкие места: раскладка текста (обрезка длинных названий,
  выравнивание колонок neofetch), подсчёт процентов языков (сумма = 100%,
  Go/Python не теряются).
- Первый прогон — визуальная проверка на реальном профиле в светлой и
  тёмной темах GitHub.
